# 二次元毛玻璃个人导航门户

一个免费、好看的个人网站导航页，动漫风毛玻璃界面、落樱粒子特效，带完整后台管理，修改内容后所有访客刷新即可看到最新内容。

**完全免费**，部署在 Cloudflare 全球网络上，国内访问速度良好。

---

## 有什么功能

**前台（访客能看到的）**
- 落樱飘落 + 星空动态背景（可在后台关闭）
- 实时时钟与随机一言（日文轻小说语录，可单独开关）
- 毛玻璃卡片展示你收藏的网站，置顶推荐卡片显示在最前并带标记
- 按分类筛选、关键词搜索
- 点卡片弹出详情窗口，一键访问或复制链接
- 展示你的头像、昵称、签名、社交联系方式
- 可显示公告横幅，访客可手动关闭
- 手机电脑都能用

**后台（你登录才能操作）**
- 添加 / 编辑 / 删除收藏的网站
- 置顶推荐网站，置顶项显示在列表最前
- 自定义网站图标（Emoji 快选或手动输入）
- 编辑每个网站的「主要功能」说明（最多 5 条）
- 管理分类标签
- 修改头像、昵称、签名、网站名称、Logo、页脚文字等基础信息
- 公告栏（后台编辑内容，前台显示可关闭横幅）
- 自定义背景图（填图片链接替换默认动态背景）
- 实时时钟、随机一言展示（可单独开关）
- 粒子特效（落樱/星空）开关
- 前台卡片点击量统计（本机记录，后台表格可查）
- 修改联系方式（GitHub、B站、邮箱、微信、QQ，留空自动隐藏）
- 直接在后台改密码，即改即生效，不用重新部署
- 一键备份 / 恢复所有数据

## 怎么搭建（一步一步来）

### 第一步：把项目复制到你的 GitHub

点击本页右上角的 **Fork** 按钮，把项目复制到你自己的账号下。

> 没有 GitHub 账号的话先去 [github.com](https://github.com) 注册一个，免费的。

---

### 第二步：注册 Cloudflare 账号

去 [cloudflare.com](https://cloudflare.com) 注册，有邮箱就能注册，不需要信用卡。

---

### 第三步：在 Cloudflare 创建项目

1. 登录 Cloudflare，左侧菜单点击 **Workers 和 Pages**
2. 点 **创建应用程序** → **Pages** → **连接到 Git**
3. 选择你刚才 Fork 的仓库
4. 构建设置填这两项：
   - **构建命令**：`node build.js`
   - **构建输出目录**：`.`（一个英文句号）
5. 点击**保存并部署**，等大约 1 分钟

---

### 第四步：创建数据库（KV）

后台数据存在 Cloudflare 的 KV 数据库里，需要先创建一个再绑定给项目。

**创建 KV：**
1. Cloudflare 左侧菜单 → **Workers 和 Pages** → **KV**
2. 点**创建命名空间**，名称填 `PORTAL_DATA`，点添加

**绑定给项目：**
1. 回到你的 Pages 项目 → **设置** → **函数** → 找到 **KV 命名空间绑定**
2. 点添加绑定：
   - 变量名称填 `PORTAL_DATA`（必须和这里一模一样）
   - KV 命名空间选刚才创建的 `PORTAL_DATA`
3. 保存

---

### 第五步：填写两个部署密钥

这两个密钥让 GitHub 自动帮你部署到 Cloudflare，只需要填一次。

在你 Fork 的 GitHub 仓库里：
1. 点顶部 **Settings** → 左侧 **Secrets and variables** → **Actions**
2. 点 **New repository secret**，添加下面两个：

| 名称 | 填什么 |
|---|---|
| `CLOUDFLARE_API_TOKEN` | Cloudflare API 令牌 |
| `CLOUDFLARE_ACCOUNT_ID` | 你的 Cloudflare 账户 ID |

**怎么拿 API 令牌：**
1. Cloudflare 右上角头像 → **我的个人资料** → **API 令牌**
2. 点**创建令牌** → 选「编辑 Cloudflare Workers」模板
3. 在权限里再加一行：账户 → **Cloudflare Pages → 编辑**
4. 创建后复制这串令牌

**怎么拿账户 ID：**
Cloudflare 右侧边栏往下滚，就能看到**账户 ID**，复制即可。

---

### 第六步：触发部署

在你的 GitHub 仓库随便改一个文件（比如在 README 末尾加个空格），提交保存。

GitHub Actions 会自动运行，大约 1 分钟部署完成。

---

### 完成！

打开 `https://你的项目名.pages.dev/admin`，系统会引导你设置管理员密码，设完直接登录，不需要做任何其他操作。

---

## 登录后台

访问 `https://你的域名/admin`

- **第一次访问**：看到"首次设置密码"界面，填两次新密码，点确认，自动登录
- **之后每次**：输入你设置的密码登录

> 连续输错 5 次会锁定 2 分钟，页面有倒计时。

---

## 关于 Logo 和浏览器标签图标

在后台 → **基础设置** → **网站 Logo 链接** 填入一个图片地址（必须是 `https://` 开头），保存后：
- 前台导航栏左上角图标变成你的图片
- 浏览器标签页图标也同步更新（包括后台页面）

留空则显示默认图标。

---

## 修改管理员密码

后台 → **修改密码**，输入当前密码和新密码，确认即可。

新密码立即同步到云端，所有设备立即生效，不需要重新部署。

## 常见问题

**Q：部署好了打开全是示例数据？**  
A：正常的。登录后台添加你自己的内容，保存后所有人刷新就能看到了。

**Q：后台保存时提示"同步失败：认证过期"？**  
A：退出后台重新登录再试。

**Q：GitHub Actions 跑失败了？**  
A：检查 GitHub Secrets 里的两个密钥有没有填错，名称区分大小写。

**Q：绑定了 KV 但后台数据存不上去（提示 503）？**  
A：KV 绑定后需要重新部署一次才生效。在 Cloudflare Pages 项目页手动点**重试部署**。

**Q：想用自己的域名？**  
A：Cloudflare Pages 项目 → **自定义域** → **设置自定义域**，填域名，按提示加一条 DNS 记录即可。

**Q：改了密码，其他设备还能用旧密码登录？**  
A：不能。改密码后旧密码立即失效，其他设备刷新登录页会自动更新。
