# 二次元毛玻璃个人导航门户

一个免费、开源的个人网站导航页，拥有动漫风毛玻璃界面、落樱粒子特效，以及完整的后台管理系统。

**核心优势：** 后台修改内容后，所有访客刷新即可看到最新数据，无需手动导出文件或重新部署。

**完全免费部署**，使用 Cloudflare Pages 全球加速。

---

## 功能介绍

### 前台展示
- 落樱飘落 + 闪烁星空动态背景
- 毛玻璃卡片展示网站集合，鼠标悬停有霓虹光晕跟随效果
- 按分类筛选、关键词搜索
- 点击卡片弹出详情窗口，展示网站简介、主要功能，一键访问或复制链接
- 展示个人资料、GitHub、邮箱、微信、QQ 等联系方式
- 完整手机端适配

### 后台管理（需密码登录）
- 添加、编辑、删除展示的网站
- 自定义网站图标：Emoji 表情快选（24 个常用图标，也可手动输入任意 Emoji）
- 编辑每个网站的「主要功能」列表（最多 5 条，详情窗口中展示）
- 管理分类标签（前台筛选栏自动更新）
- **基础设置**：修改网站名称、昵称、身份标签、签名、头像链接、版权署名、**网站 Logo**、**页脚副标题**
- 修改联系方式（GitHub、邮箱、微信、QQ）
- **直接修改管理员密码**：后台修改后立即同步到云端，所有设备生效，无需重新部署
- **一键备份/恢复**：将全部数据导出为 JSON，或从备份文件一键恢复

## 如何搭建（完整步骤）

### 第一步：复制项目到你的 GitHub

点击页面右上角的 **Fork** 按钮，把这个项目复制到你自己的 GitHub 账户。

### 第二步：注册 Cloudflare 账号

前往 [cloudflare.com](https://cloudflare.com) 注册免费账号（有邮箱即可，无需信用卡）。

### 第三步：在 Cloudflare 创建网站项目

1. 登录 Cloudflare → 左侧菜单点击 **Workers 和 Pages**
2. 点击 **创建应用程序** → **Pages** → **连接到 Git**
3. 选择你 Fork 的仓库，点击 **开始设置**
4. 构建配置填写如下：
   - **构建命令**：`node build.js`
   - **构建输出目录**：`.`（一个英文句号）
5. 点击**保存并部署**，等待第一次部署完成（约 1 分钟）

### 第四步：创建数据存储空间（KV）

后台数据（网站列表、个人设置、密码等）存储在 Cloudflare KV 数据库中。

1. Cloudflare 左侧菜单 → **Workers 和 Pages** → **KV**
2. 点击**创建命名空间**，名称填 `PORTAL_DATA`，点击添加

然后绑定到你的网站项目：

1. 回到 Pages 项目 → **设置** → **函数** → 找到 **KV 命名空间绑定**
2. 点击添加绑定：
   - **变量名称**：`PORTAL_DATA`（必须完全一致）
   - **KV 命名空间**：选择刚才创建的 `PORTAL_DATA`
3. 保存

### 第五步：配置 GitHub Secrets（自动部署必需）

在你 Fork 的 GitHub 仓库中：

1. 点击仓库顶部 **Settings** → 左侧 **Secrets and variables** → **Actions**
2. 点击 **New repository secret**，依次添加以下 2 个：

| 名称 | 填写什么 |
|---|---|
| `CLOUDFLARE_API_TOKEN` | Cloudflare API 令牌（见下方说明） |
| `CLOUDFLARE_ACCOUNT_ID` | 你的 Cloudflare 账户 ID（见下方说明） |

**获取 Cloudflare API 令牌：**
1. Cloudflare → 右上角头像 → **我的个人资料** → **API 令牌**
2. 点击**创建令牌** → 使用「编辑 Cloudflare Workers」模板
3. 在权限列表中额外添加：账户 → **Cloudflare Pages → 编辑**
4. 点击创建，复制令牌

**获取账户 ID：**
Cloudflare 任意页面 → 右侧边栏往下找 → 复制**账户 ID**

### 第六步：触发部署

在你的 GitHub 仓库随便编辑一个文件（比如在 README 末尾加个空格），提交保存。

GitHub Actions 自动运行，约 1 分钟完成部署。

**完成！** 访问 `https://你的项目名.pages.dev/admin`，首次部署时系统会引导你直接设置管理员密码，无需去 Cloudflare 手动操作。

## 登录后台

访问 `https://你的域名/admin`。

- **首次部署**：系统检测到没有密码时，自动显示"首次设置密码"界面，直接设置即可登录，无需任何额外操作。
- **之后登录**：输入你设置的密码即可。

登录后所有修改都会**自动同步**到云端，其他人刷新即可看到最新内容。

> 连续输错密码 5 次会触发 2 分钟锁定保护，页面有倒计时提示。

## 自定义内容

### 基础设置（后台直接修改）

登录后台 → **基础设置**，可修改：

| 设置项 | 说明 |
|---|---|
| 网站名称 | 显示在浏览器标签和导航栏 |
| 作者名称 | 个人资料卡的昵称 |
| 身份标签 | 昵称旁边的标签，如「独立开发者」 |
| 个性签名 | 个人资料卡下方的一句话 |
| 头像图片链接 | 留空使用默认动漫头像 |
| 版权署名 | 页脚版权行的名字 |
| 网站 Logo 链接 | 导航栏左上角图标，同步更新浏览器标签图标；留空使用默认图标 |
| 页脚副标题 | 「由❤名字」后面的文字，如「开发·免费托管于 Cloudflare Pages」 |

### 修改默认初始数据

编辑 `js/data.js` 文件中的 `initialWebsitesData` 数组，这是 KV 数据库为空时（首次部署前）展示的默认网站列表。

## 修改管理员密码

登录后台 → **修改密码**，输入当前密码和新密码，点击确认。

新密码立即同步到云端，所有设备登录均使用新密码，**无需重新部署**。

## 常见问题

**Q：部署后打开是示例数据，不是我的内容？**  
A：正常现象。示例数据来自 `js/data.js`，登录后台添加内容保存后即可看到你的数据。

**Q：后台保存时提示「云端同步失败：认证过期」？**  
A：退出后台重新登录再试。如果反复出现，进入「修改密码」页面重新设置一次密码。

**Q：推送代码后 GitHub Actions 报错？**  
A：检查 GitHub Secrets 中的 `CLOUDFLARE_API_TOKEN` 和 `CLOUDFLARE_ACCOUNT_ID` 是否填写正确，名称区分大小写。

**Q：KV 绑定了但后台数据存不上去（503 错误）？**  
A：KV 绑定后需重新部署才生效。在 Cloudflare Pages 项目页面点击**重试部署**，或推送一次代码触发重新部署。

**Q：想绑定自己的域名怎么做？**  
A：Cloudflare Pages 项目 → **自定义域** → **设置自定义域**，填写域名，按提示在 DNS 添加一条记录即可。

**Q：修改密码后其他设备还是用旧密码能登录？**  
A：密码修改会立即写入 KV 云端，其他设备刷新登录页会自动获取新密码哈希，旧密码立即失效。

**Q：Logo 图片设置后没有更新？**  
A：确认填写的是完整的 `https://` 开头的图片链接，图片需允许跨域加载。保存后刷新页面生效。
